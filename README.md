### Symfony 6.3 Messenger issue

1. Have an external transport which does not have any metadata like `BusNameStamp`
2. Force bus name in `messenger:consume` command with `--bus`
3. Have message handler which dispatches another message, with `DispatchAfterCurrentBusStamp`
4. External message handler finishes successfully
5. Delayed message from step 3 throws an error
6. Messenger will retry the message but it will be missing `BusNameStamp` and all other stamps from previous middlewares (like `HandledStamp`)


### Possible fixes

See [the pull request](https://github.com/symfony/symfony/pull/51468)
